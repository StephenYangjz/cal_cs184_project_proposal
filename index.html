Avalanche Simulation

By Prince Wang, Stephen Yang, Rory Na, Christian Kim

General Description:

We aim to simulate an avalanche on a mountain. Ideally, the final deliverable involves a vivid 360 view of snow rolling off a mountain range, with simulated fog rising up from the snow.  

Problem Description:

Inspired by the beautiful snow scenes in the Beijing 2022 winter Olympics, we want to create a vivid and realistic rendering of an avalanche. On one hand, it is useful as a visual and artistic project that provides related artistic industries techniques on how to render beautiful snow scenes; on the other hand, it could be used to measure damage potentials on where avalanches could really happen. For instance, in 2020, avalanches cause 37 casualties in the U.S. alone, so with a better way to simulate when, how, and why avalanches happen, we can have a more concrete sense of how to measure and prevent the damages associated with avalanches.

The project would be challenging because a good avalanche simulation has many parts to it, involving realistic snow simulation with many parameters to tune, fog simulation that comes from the snow, and also a GUI that can be used to control parameters involving where and how the avalanche happens.

Goals and Deliverables:

We plan to implement a basic interaction system where a user is able to observe snow falling down a mountain. With this plan in mind, our final goal is to produce a video demo where you could simulate snowballs falling down a steep mountain, with tunable parameters such as where/what height the fall is starting. This means we will also need to implement a User Interface. We believe this is an achievable demo as we have a few papers and tutorials outlying the basic formulas that build up this interactive system (see our resource section). A feature we aim to implement is real-time rendering on GPU.

If things go really well, then we will shoot for simulating an avalanche, which is a lot more sophisticated than snowballs falling. It probably requires way more effort and is a lot harder to render. In terms measure the quality of our simulation, it is difficult to have a quantitative metric for such simulation, so likely we will just be eye-balling how well the simulation looks. For more accurate simulation, we will probably compare our results with videos of real-world avalanche/ snowball falling.

Schedule:

First Week: Creating baseline (snow-like) particle simulations

Second Week: Turn the model to support “snowball falling” scenes.

Third Week: Creating working avalanche simulations, and other related elements.

Fourth Week: Add some level of intractability (starting disturbance point in the snow, level of disturbance, etc), and if we still have time we can probably do fog simulation.

Resources:

We will reference a variety of available materials, such as academic papers, online articles and past CS184 projects. We will probably use C++ and/or OpenGL in our implementation. If simulation proves to be computationally expensive, we will incorporate CUDA acceleration if possible.
